About This Application

The Application

The application makes use of Heroku, an open source PaaS, with a Bootstrap framework. The CSS and HTML were customized for simplicity, but maintain the connectivity between the various components (using Flask). All documents are maintained on Github (see link above).

In order for the flask application to work, we needed to provide the (1) model, our Decision Tree Regressor, (2) the fitted StandardScaler and (3) supporting data to the application. These needed to be compiled and used. The specific python packages that were used in code development in the jupyter notebook (on GitHub, see above), are also provided in requirement.txt for the Heroku application to use.

The directory layout includes Templates and Static, with these key files procfile and app.py. The templates directory contains the HTML framework for each site, and the appropriate code to interact with the app.py (which loads data, model artifacts, transforms data using python, predicts, and renders/returns the site as designed.) The Static directory contains statics files, images, models, and data used in the application or elsewhere. Procfile is a supporting file instructing Heroku which application (here app.py) to use. All are uploaded to Heroku in which this application is hosted.

Data Science Workflow

The general steps for the Data Science Workflow is listed below, and was iterative in nature, as more was learned throughout the process. In GitHub (see link above), the jupyter notebook shows all code, object creation, and fully discusses this project.

  1. Retrieve Data
  2. Exploratory Data Analysis: Clean & Explore
  3. Prepare & Transform
  4. Develop & Train Model(s)
  5. Validate & Evaluate Results
  6. Deployment & Objects

The final model chosen needed to generalize well to unseen data. Even though the earlier models performed well on the training data (accuracy ~95%), the test accuracy was much lower (50%). This indicates overfitting and poor generalization, i.e. our new prediction in the application would not be that great.

So, the final model chosen was a Decision Tree Regression, trained using cross-validation and grid searching of hyperparameters. This resulted in a lower training accuracy of 81.5% but a much closer (and higher) test accuracy of 77.2%. The closeness, ~5% indicates that the model is generalizing well to unseen data. As a result, the best estimator was used, saved, and deployed. Further improvement and next steps is discussed in the jupyter notebook.
